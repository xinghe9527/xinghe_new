## 功能概述
已成功实现用户修改密码功能，符合 PocketBase 官方安全规范。精简了用户菜单，只保留核心的密码修改功能。

- ❌ 删除"编辑个人资料"菜单项（不再支持修改头像和用户名）
- ✅ 保留"账号设置"菜单项，副标题改为"修改密码"
- ✅ 保留"退出登录"菜单项

## 实现细节

### 1. API 层 (`lib/features/auth/data/auth_api_service.dart`)
新增 `updatePassword` 方法：
- 使用 PATCH 请求访问 `$baseUrl/api/collections/users/records/$userId`
- 请求载荷包含三个必需字段：
  - `oldPassword`: 用户当前密码
  - `password`: 新密码
  - `passwordConfirm`: 新密码确认（与 password 相同）
- 携带 Authorization token 进行身份验证

### 2. Provider 层 (`lib/features/auth/presentation/auth_provider.dart`)
新增 `updatePassword` 方法：
- 验证用户登录状态
- 调用 API 服务修改密码
- 统一错误处理

### 3. UI 层 (`lib/features/auth/presentation/widgets/user_header_widget.dart`)
优化 `_AccountSettingsDialog` 对话框：
- 三个密码输入框：当前密码、新密码、确认新密码
- 完整的输入验证：
  - 检查所有字段是否填写
  - 验证新密码长度（至少8个字符）
  - 确认两次输入的新密码一致
- 加载状态显示（按钮显示加载动画）
- **错误提示显示在输入框下方**（与注册页面一致的风格）
- 修改成功后自动退出登录，要求用户重新登录

## 用户交互流程

1. 用户点击头像 → 选择"账号设置"
2. 在弹出的对话框中输入：
   - 当前密码
   - 新密码（至少8个字符）
   - 确认新密码
3. 点击"确认修改"按钮
4. 系统验证输入并调用 API
5. 如果有错误：
   - **错误信息显示在输入框下方的红色提示框中**
   - 用户可以直接修改输入，无需关闭对话框
6. 修改成功后：
   - 显示"密码修改成功，请重新登录"提示（SnackBar）
   - 1秒后自动退出登录
   - 用户需要使用新密码重新登录

## 安全特性

- 符合 PocketBase 官方密码修改规范
- 必须提供旧密码验证身份
- 新密码需要二次确认
- 修改成功后强制重新登录，刷新 Token
- 所有密码输入框使用 `obscureText: true` 隐藏输入内容

## 错误处理

所有错误信息均已本地化为中文：

### 客户端验证错误（立即显示）
- 当前密码为空：显示"请输入当前密码"
- 新密码为空：显示"请输入新密码"
- 新密码长度不足：显示"新密码至少需要8个字符"
- 两次密码不一致：显示"两次输入的新密码不一致"

### 服务器验证错误（API 返回）
- 当前密码错误：显示"当前密码错误"
- 新密码格式不正确：显示"新密码格式不正确"
- 新密码过短：显示"新密码至少需要8个字符"
- 密码确认不匹配：显示"两次输入的新密码不一致"
- 网络超时：显示"修改密码失败，请稍后重试"
- 其他错误：显示"修改密码失败，请检查输入信息"

### 错误信息解析逻辑
API 层会智能解析 PocketBase 返回的英文错误信息，自动转换为中文：
- 检查 HTTP 状态码（401/403 → 密码错误）
- 解析 message 字段关键词
- 解析 data 字段中的具体错误代码
- 提供友好的中文错误提示
