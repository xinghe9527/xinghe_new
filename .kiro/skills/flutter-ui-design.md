# Flutter UI 设计最佳实践

这个 skill 提供 Flutter UI 开发的最佳实践和设计指导。

## 性能优化

- 尽可能使用 `const` 构造函数来减少重建
- 避免在 `build` 方法中创建新对象
- 使用 `ListView.builder` 而不是 `ListView` 处理长列表
- 合理使用 `RepaintBoundary` 隔离重绘区域

## 代码组织

- 将复杂的 Widget 提取为独立的类或方法
- 使用有意义的变量名和方法名
- 保持 Widget 树的扁平化，避免过深嵌套
- 将样式常量提取到单独的文件或类中

## 主题和样式

- 使用 `Theme.of(context)` 获取主题颜色
- 定义统一的颜色常量，避免硬编码
- 使用 `TextTheme` 统一文本样式
- 考虑深色模式支持

## 响应式设计

- 使用 `MediaQuery` 获取屏幕尺寸
- 使用 `LayoutBuilder` 根据可用空间调整布局
- 考虑不同屏幕尺寸的适配
- 使用 `Flexible` 和 `Expanded` 实现弹性布局

## 用户体验

- 提供加载状态指示器
- 处理错误状态并显示友好的错误信息
- 使用动画提升交互体验（但不要过度）
- 确保触摸目标足够大（至少 48x48）

## 可访问性

- 为图标和图片提供语义标签
- 确保足够的颜色对比度
- 支持屏幕阅读器
- 提供键盘导航支持
