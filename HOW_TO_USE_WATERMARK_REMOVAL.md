# 如何使用水印去除功能

## 🎯 给开发者的说明

### 第一次使用前的准备

1. **下载模型文件**（只需一次）
   ```bash
   cd python_backend
   python download_model.py
   ```
   这会下载 `lama_model.onnx` 文件（约 50MB）到 `python_backend` 目录。

2. **打包引擎**（只需一次，或更新时）
   
   Windows:
   ```bash
   cd python_backend
   build_engine.bat
   ```
   
   Linux/Mac:
   ```bash
   cd python_backend
   chmod +x build_engine.sh
   ./build_engine.sh
   ```
   
   这会生成 `watermark_engine.exe`（或 `watermark_engine`）文件。

3. **测试引擎**（可选，建议）
   ```bash
   # 启动引擎
   cd python_backend
   watermark_engine.exe  # Windows
   ./watermark_engine    # Linux/Mac
   
   # 在另一个终端测试
   python test_engine.py
   ```
   
   如果看到 "✅ 引擎运行正常！"，说明一切正常。

### 开发环境运行

直接运行 Flutter 应用即可：

```bash
flutter run
```

应用会自动：
1. 在启动时启动水印引擎（后台静默运行）
2. 等待引擎就绪（2-5 秒）
3. 在退出时自动清理引擎进程

你会在控制台看到类似的日志：

```
🚀 正在启动水印去除引擎...
📂 引擎路径: C:\...\watermark_engine.exe
🔧 引擎进程已启动 (PID: 12345)
⏳ 等待引擎就绪...
✅ 引擎已就绪 (耗时: 3 秒)
✅ 水印去除引擎已启动
```

### 发布应用

1. **构建 Flutter 应用**
   ```bash
   flutter build windows --release
   ```

2. **复制引擎文件**
   
   将以下文件复制到 `build/windows/runner/Release/` 目录：
   - `python_backend/watermark_engine.exe`
   - `python_backend/lama_model.onnx`
   
   最终目录结构：
   ```
   build/windows/runner/Release/
   ├── xinghe_new.exe          # 你的应用
   ├── watermark_engine.exe    # 水印引擎
   ├── lama_model.onnx         # 模型文件
   └── ... (其他 Flutter 文件)
   ```

3. **测试发布版本**
   
   直接运行 `xinghe_new.exe`，测试水印去除功能是否正常。

4. **创建安装包**（可选）
   
   使用 Inno Setup 或 NSIS 创建安装包时，确保包含上述三个文件。

## 🎨 给用户的使用说明

### 打开水印去除工具

1. 启动应用
2. 点击左侧菜单的"工具箱"
3. 点击"图片去水印"

### 选择图片

点击"选择图片"按钮，可以：
- 选择单张图片
- 选择多张图片（批量处理）

支持的格式：JPG、PNG

### 标记水印区域

有三种方式标记水印：

#### 1. 智能检测（推荐）
- 选择"智能检测"工具
- 调整"检测灵敏度"滑块
- 点击"处理当前"
- 系统会自动识别并去除水印

#### 2. 手动涂抹
- 选择"手动涂抹"工具
- 调整"画笔大小"滑块
- 在图片上涂抹水印区域（会显示青绿色痕迹）
- 点击"处理当前"

#### 3. 矩形框选
- 选择"矩形框选"工具
- 在图片上拖动鼠标框选水印区域
- 可以框选多个区域
- 点击"处理当前"

### 查看结果

处理完成后：
- 图片会自动显示处理结果
- 点击"对比原图"可以左右对比原图和处理后的图片
- 如果效果不满意，点击"清除标记"重新标记

### 批量处理

如果选择了多张图片：
1. 标记第一张图片的水印区域
2. 点击"批量处理"
3. 系统会用相同的方式处理所有图片
4. 使用左右箭头切换查看不同图片的结果

### 保存结果

点击"保存结果"按钮：
- 处理后的图片会保存到原图所在目录的 `watermark_removed` 子文件夹
- 文件名格式：`原文件名_removed.jpg`

## ⚠️ 注意事项

### 首次使用
- 首次启动应用时，引擎需要 2-5 秒加载模型
- 请耐心等待，不要重复点击

### 处理速度
- CPU 处理：约 5-10 秒/张（1920x1080）
- GPU 处理：约 1-2 秒/张（1920x1080）
- 图片越大，处理时间越长

### 最佳实践
1. **智能检测**适合：水印位置固定、对比度高的情况
2. **手动涂抹**适合：水印形状不规则、需要精确控制的情况
3. **矩形框选**适合：水印是规则矩形、位置明确的情况

### 常见问题

**Q: 为什么处理速度很慢？**  
A: 这是正常的，CPU 处理大图片需要时间。如果有 NVIDIA GPU，会自动加速。

**Q: 处理效果不好怎么办？**  
A: 尝试：
- 调整检测灵敏度
- 使用手动涂抹精确标记
- 确保标记区域完全覆盖水印

**Q: 可以处理视频吗？**  
A: 目前只支持图片，视频功能正在开发中。

**Q: 处理后的图片质量会下降吗？**  
A: 不会。处理后的图片保持原始分辨率和质量。

**Q: 需要联网吗？**  
A: 不需要。所有处理都在本地完成，保护您的隐私。

## 🔧 故障排查

### 引擎启动失败

如果看到"引擎启动失败"的提示：

1. 检查应用目录下是否有这两个文件：
   - `watermark_engine.exe`
   - `lama_model.onnx`

2. 检查端口 8000 是否被占用：
   - Windows: 打开任务管理器，结束占用端口的程序
   - 或重启电脑

3. 检查防火墙设置：
   - 允许应用访问本地网络（127.0.0.1）

### 处理失败

如果处理失败：

1. 确保图片格式正确（JPG 或 PNG）
2. 确保图片没有损坏
3. 尝试重新标记水印区域
4. 重启应用

### 应用卡死

如果应用无响应：

1. 等待处理完成（大图片需要较长时间）
2. 如果长时间无响应，关闭应用重新打开
3. 尝试处理较小的图片
