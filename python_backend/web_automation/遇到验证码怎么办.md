# 🤖 遇到机器人验证怎么办？

## 📌 现在的改进

### ✅ 1. 浏览器不会自动关闭了

**之前的问题**:
- 遇到错误就关闭浏览器
- 你看不到具体发生了什么

**现在的改进**:
- ✅ 遇到错误时，浏览器会保持打开
- ✅ 你可以看到具体的页面状态
- ✅ 你可以手动完成验证
- ✅ 按 Ctrl+C 才会关闭浏览器

### ✅ 2. 添加了反机器人检测功能

**新增的反检测特性**:
- ✅ 隐藏 webdriver 特征
- ✅ 伪装浏览器指纹
- ✅ 随机操作延迟（模拟人类）
- ✅ 随机鼠标移动（模拟人类）
- ✅ 模拟人类打字速度（50-100ms/字符）
- ✅ 随机页面滚动

## 🎯 遇到验证码时的操作步骤

### 步骤 1: 不要慌张

当你看到验证码弹窗时：
```
✅ 浏览器会保持打开
✅ 脚本会暂停（不会关闭）
✅ 你有充足的时间处理
```

### 步骤 2: 手动完成验证

常见的验证类型：

1. **图片验证**
   - 点击指定的图片
   - 例如："请点击所有包含交通灯的图片"

2. **滑块验证**
   - 拖动滑块到指定位置
   - 完成拼图

3. **文字验证**
   - 输入图片中的文字或数字

### 步骤 3: 验证完成后

**如果脚本还在运行**:
- ✅ 验证通过后，页面会自动继续
- ✅ 脚本会继续执行后续操作
- ✅ 不需要你做任何额外操作

**如果脚本已经退出**:
- ✅ 重新运行脚本
- ✅ 由于使用持久化登录，验证状态会保留
- ✅ 下次可能不会再遇到验证

### 步骤 4: 如果验证失败

1. **刷新页面重试**
   - 在浏览器中按 F5
   - 重新完成验证

2. **重新运行脚本**
   ```bash
   # 按 Ctrl+C 退出当前脚本
   # 然后重新运行
   python api_server.py
   ```

3. **手动登录一次**
   ```bash
   python python_backend/web_automation/init_login.py vidu
   ```

## 🛡️ 如何减少遇到验证码？

### 方法 1: 降低操作频率

**不要这样做** ❌:
```
连续生成 10 个视频
每个视频间隔 10 秒
```

**应该这样做** ✅:
```
生成 1-2 个视频
等待 5-10 分钟
再生成下一批
```

### 方法 2: 模拟正常用户

**不要这样做** ❌:
```
打开网站 → 立即生成 → 关闭
```

**应该这样做** ✅:
```
打开网站 → 浏览一下 → 等待几秒 → 生成 → 查看结果
```

### 方法 3: 保持账号活跃

**建议**:
- 定期手动登录网站
- 手动生成几个视频
- 查看历史记录
- 让账号看起来像真实用户

### 方法 4: 使用已验证的账号

**首次使用时**:
1. 手动登录网站
2. 完成所有初始验证
3. 手动生成 1-2 个视频
4. 然后再使用自动化脚本

## 📊 验证码出现的原因

### 常见触发条件

1. **操作太快**
   - 连续快速点击
   - 没有等待时间

2. **操作太规律**
   - 每次间隔完全相同
   - 操作顺序完全相同

3. **新账号或新 IP**
   - 刚注册的账号
   - 第一次使用的 IP 地址

4. **异常行为**
   - 短时间内大量操作
   - 与正常用户行为不符

### 我们的应对措施

✅ **已实现**:
- 随机延迟（0.5-2 秒）
- 随机鼠标移动
- 模拟人类打字速度
- 随机页面滚动
- 使用持久化登录

## 🔧 调试技巧

### 查看浏览器控制台

当浏览器保持打开时：

1. **按 F12** 打开开发者工具
2. **查看 Console** 标签页
   - 看是否有错误信息
   - 看是否有验证相关的提示

3. **查看 Network** 标签页
   - 看是否有失败的请求
   - 看是否有验证相关的请求

### 查看脚本日志

脚本会输出详细的日志：

```
🔍 查找可见的输入框...
✅ 找到可见输入框: textarea:visible
📝 开始填充提示词...
🎯 暴力聚焦：强制夺取输入框焦点...
✅ 焦点已强制夺取
⌨️  输入提示词: 一个赛博朋克风格的女孩
✅ 提示词填充成功
```

如果看到错误，会显示：

```
❌ 执行失败: [错误信息]

⚠️  发生错误，浏览器将保持打开状态
请手动检查页面，查看具体问题
按 Ctrl+C 可以退出脚本
```

## 💡 实用建议

### 首次使用

1. **先手动测试**
   ```bash
   # 手动登录并生成一个视频
   python python_backend/web_automation/init_login.py vidu
   ```

2. **然后自动化**
   ```bash
   # 启动 API 服务器
   python api_server.py
   
   # 在 Flutter 中生成视频
   ```

### 日常使用

1. **合理安排时间**
   - 早上生成几个
   - 下午生成几个
   - 不要集中在短时间内

2. **观察反馈**
   - 如果经常遇到验证，说明频率太高
   - 适当增加间隔时间

3. **保持耐心**
   - 自动化是为了提高效率
   - 不是为了滥用服务
   - 合理使用才能长期使用

### 遇到问题

1. **浏览器不关闭了**
   - 你可以看到具体问题
   - 你可以手动完成验证
   - 你可以截图保存错误信息

2. **查看日志**
   - 脚本会输出详细的步骤
   - 可以看到在哪一步出错

3. **寻求帮助**
   - 截图保存错误信息
   - 记录操作步骤
   - 查看文档或寻求支持
