# ✨ 资产继承功能

## 功能概述

支持在创建新作品时继承现有作品的所有资产（角色、场景、物品），方便创作续集或相关作品。

## 功能特性

### 1. 创建作品时选择继承源

在创建新作品对话框中，可以选择从哪个作品继承资产：

- **不继承（从零开始）**：默认选项，创建全新作品
- **选择现有作品**：从下拉列表中选择要继承的作品

### 2. 资产视觉标识

继承的资产在列表中有明显的视觉区分：

- **继承的资产**：
  - 名称显示为**白色**
  - 左侧有文件夹图标 📁
  
- **新创建的资产**：
  - 名称显示为**灰色**
  - 无特殊图标

### 3. 自动复制所有资产

选择继承源后，系统会自动复制：
- ✅ 所有角色（包括图片、描述、映射代码）
- ✅ 所有场景（包括图片、描述、映射代码）
- ✅ 所有物品（包括图片、描述、映射代码）

### 4. 独立管理

继承后的资产完全独立：
- 可以删除任何资产（包括继承的）
- 可以修改资产的描述
- 可以重新生成资产的图片
- 对继承资产的修改不影响源作品

## 使用场景

### 场景 1：续集创作

```
作品 17（第一集）
├─ 角色：小明、小红、小刚
├─ 场景：学校、公园、家
└─ 物品：书包、足球

创建作品 22（第二集）
├─ 继承自：作品 17
├─ 自动获得：小明、小红、小刚、学校、公园、家、书包、足球
└─ 新增：小李（新角色）、图书馆（新场景）
```

### 场景 2：相关作品

```
作品 A（主线故事）
├─ 角色：主角、配角A、配角B
└─ 场景：城市、森林

创建作品 B（支线故事）
├─ 继承自：作品 A
├─ 保留：主角、配角A、城市
├─ 删除：配角B、森林（不需要）
└─ 新增：配角C（新角色）、海边（新场景）
```

## 技术实现

### 数据结构

所有资产数据模型都添加了两个字段：

```dart
class CharacterData {
  final bool isInherited;      // 是否继承
  final String? sourceWorkId;  // 来源作品ID
  // ...
}
```

### 继承逻辑

```dart
Future<void> _inheritAssets(String sourceWorkId, String targetWorkId) async {
  // 1. 读取源作品的资产数据
  // 2. 为每个资产生成新ID
  // 3. 标记 isInherited = true
```

## 使用建议

1. **续集创作**：直接继承前作的所有资产，删除不需要的，添加新的
2. **相关作品**：继承共同的角色和场景，各自添加独特的元素
3. **实验版本**：继承主版本的资产，在新作品中尝试不同的创作方向
4. **资产管理**：通过继承功能，可以快速复用已有的高质量资产

## 注意事项

1. **独立性**：继承后的资产完全独立，修改不影响源作品
2. **存储空间**：继承会复制图片文件，注意磁盘空间
3. **命名规范**：确保资产名称统一，方便识别和管理
4. **定期清理**：删除不需要的资产，保持作品整洁
